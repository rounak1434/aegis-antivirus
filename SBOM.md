# Software Bill of Materials (SBOM)

Every Aegis release ships an SBOM + supply-chain inventories so consumers can
audit exactly what's inside.

## What's produced

| File | Format | Contents |
|------|--------|----------|
| `sbom.cargo.cdx.json` | CycloneDX 1.5 JSON | every Rust crate in the workspace (name, version, `pkg:cargo` PURL, license) |
| `sbom.npm.cdx.json` | CycloneDX / npm list JSON | frontend dependency graph |
| `license-inventory.csv` | CSV | name, version, license, repository for all crates |
| `dependency-tree.txt` | text | `cargo tree` of the workspace |

Generated by `deploy/generate-sbom.ps1` (also invoked by `make-release.ps1`).

## How it's generated

```powershell
deploy\generate-sbom.ps1 -OutDir .\dist
```

- **Rust:** prefers `cargo cyclonedx` if installed; otherwise builds a CycloneDX
  document from `cargo metadata` (a `bomFormat`/`specVersion`/`components`
  object with one `library` component per crate). This guarantees an SBOM even
  offline.
- **Frontend:** prefers `cyclonedx-npm`; otherwise emits `npm ls --all --json`.
- **Inventories:** license CSV + `cargo tree` from `cargo metadata`.

## Verified locally

A real run in this repo produced:

- `sbom.cargo.cdx.json` — **CycloneDX 1.5, 726 components**
- `license-inventory.csv` — 727 rows
- `dependency-tree.txt` — 1,405 lines

## Consuming the SBOM

CycloneDX JSON is consumable by Dependency-Track, `grype`, `trivy sbom`, OSV
scanners, etc. Example:

```bash
grype sbom:./sbom.cargo.cdx.json        # vulnerability scan from the SBOM
trivy sbom ./sbom.cargo.cdx.json
```

The SBOM complements the CI security gate (`cargo-deny` / `cargo-audit` /
dependency review) — the gate blocks bad dependencies at PR time; the SBOM
records exactly what shipped.

## Notes

- For the richest CycloneDX output (hashes, full license expressions, dependency
  edges), install `cargo-cyclonedx` and `cyclonedx-npm` before generating; the
  release CI does this best-effort.
- SPDX can be emitted instead/as-well via `cargo sbom` or `syft` if a consumer
  requires SPDX — the release scripts standardize on CycloneDX.
